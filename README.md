## 1. 투포인터, 슬라이딩 윈도우 문제
**문제 설명**  
1110001110111 과 같은 배열이 주어지고, 0을 채울 수 있는 횟수(n)가 정해져 있음. 연속되는 최대 1의 크기를 구하는 문제.  
n의 크기는 최대 3 * 10^5.

**Point**
- 조합으로 풀 수 있으나, n의 크기로 인해서 투포인터, 슬라이딩 윈도우로 풀어야하는 문제  
  
출처 : 현대카드

## 2. 완전탐색
**문제 설명**  
2개의 가스통을 채울 수 있는 3가지 조건이 있고, 2개의 가스통에 채워질 수 있는 물의 양을 오름차순으로 정렬하여 배열을 리턴해야함.  

**Point**  
처음에 문제를 이해못했으나, 6가지 조건에 따라 완전 탐색을 통해 가스통에 채워지는 물의 양을 기록하면 된다(각 가스통마다 3가지 조건)  
  
출처 : 현대카드

## 3. 연결리스트 문제

이중연결리스트를 구현해서 풀어야 하는 문제.  
주로 중간 Element를 삽입, 삭제하는 시간에서 이득을 많이 볼 수 있음(Time Complexity : O(1)).  
배열의 경우, 중간 Element를 삽입하거나 삭제할 때, O(n)의 시간이 소요됨(삽입/삭제된 위치 다음의 나머지 Element들을 한칸씩 뒤/앞으로 복사하는 시간)  
  
구현할 때, head, tail은 DummyNode로 생성하는 것이 구현할 때 훨씬 쉽다.  

문제 : Leetcode LRU.
